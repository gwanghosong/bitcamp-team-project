# UC001 - 회원 가입(Sign Up)
비 회원이 시스템을 이용하기 위해 회원을 가입하는 유스케이스

## 주 액터(Primary Actor)
비회원

## 보조 액터(Secondary Actor)
페이스북, 구글, 카카오

## 사전 조건(Preconditions)
- 회원에 가입하지 않은 상태이다.
- 회원 가입시에 입력하는 아이디가 시스템에 등록되어 있지 않다.

## 종료 조건(Postconditions)
- 회원 정보가 시스템에 등록되었다.

## 시나리오(Flow of Events)

### 기본 흐름(Basic Flow of Events)
- 1. 비회원이 회원 가입 버튼을 클릭할 때 이 유스케이스를 시작한다.
- 2. 시스템은 약관 동의 폼을 출력한다.
- 3. 액터는 서비스이용약관, 개인정보 수집 및 이용동의, 개인정보 제3자 제공 동의, 마케팅 및 광고에의 활용에 관한 체크박스를 체크하여 동의한 후 다음 버튼을 클릭한다.
- 4. 시스템은 회원 정보 입력 폼을 출력한다.
- 5. 액터는 아이디, 이름, 비밀번호, 비밀번호재확인, 생년월일, 성별, 휴대전화, 이메일, 프로필사진, 주소, 자기소개를 입력한 후 가입신청 버튼을 클릭한다.
- 6. 시스템은 회원 정보를 저장한 후 가입완료 폼을 출력한다.
- 7. 액터가 메인화면 버튼을 클릭하면 '메인화면' 유스케이스로 가고, 로그인 버튼을 클릭하면 '로그인' 유스케이스로 간다.


### 대안 흐름(Alternative Flows)

- 3.1 액터가 페이스북 버튼을 클릭하면,
    - 시스템은 '페이스북 아이디로 회원 가입하기' 유스케이스로 간다.
- 3.2 액터가 구글 버튼을 클릭하면,
    - 시스템은 '구글 아이디로 회원 가입하기' 유스케이스로 간다.
- 3.3 액터가 카카오 버튼을 클릭하면,
    - 시스템은 '카카오 아이디로 회원 가입하기' 유스케이스로 간다.

### 예외 흐름(Exception Flows)
- 4.1 서비스 이용약관, 개인정보 수집 및 이용동의, 개인정보 제 3자 제공동의 체크박스를 체크하지 않으면,
    - 시스템은 필수 동의 약관에 동의해야 함을 알리는 내용을 출력한다.
- 4.2 아이디, 비밀번호, 비밀번호 재확인, 이름, 생년월일, 성별, 휴대전화, 이메일, 주소/상세주소 중에 한 개라도 값이 비어 있으면,
    - 시스템은 필수 입력 항목이 비어 있음을 알리는 내용을 출력한다.
- 4.3 아이디, 이메일이 이미 등록되어 있다면,
    - 시스템은 중복된 이메일임을 알리는 내용을 출력한다.
- 4.4 암호와 재확인 암호가 일치하지 않으면,
    - 시스템은 입력된 암호가 일치하지 않음을 알린다.
